# 🚖 Ride Fare Surge Analysis - Insight Summary

## Project Objective

The objective of this project was to analyze ride booking data to understand booking trends, revenue patterns, surge pricing, customer behavior, and operational performance. Python was used for data cleaning, feature engineering, and exploratory data analysis (EDA), while Power BI was used to build an interactive business dashboard.


## Dataset Overview

- Raw ride booking dataset
- Data cleaned and transformed using Python
- Feature engineering performed for better analysis
- Cleaned dataset used in Power BI


## Key Business Insights

### Booking Performance
- Most ride bookings were completed successfully.
- Customer and driver cancellations accounted for a smaller percentage of total bookings.

### Revenue Analysis
- Revenue differed across vehicle categories.
- Premium vehicle types generated higher revenue per ride.
- Peak hours contributed significantly to total revenue.

### Surge Pricing
- Surge pricing was observed mainly during high-demand periods.
- Booking volume increased during surge hours.

### Customer Experience
- Average customer ratings remained above 4.
- UPI was the most preferred payment method.

### Operational Insights
- Some pickup and drop locations consistently showed higher ride demand.
- Driver cancellation reasons highlighted operational challenges such as long pickup distances.

## Recommendations

- Improve driver availability during peak hours.
- Optimize driver allocation for high-demand locations.
- Reduce pickup distance to minimize driver cancellations.
- Monitor surge pricing to maintain customer satisfaction.
